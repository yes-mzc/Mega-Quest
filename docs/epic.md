# Epic PRD: Mega-Quest MVP — 모바일 기반 온보딩 통합 에이전트

---

## 1. Epic Name

**Mega-Quest MVP: 게이미피케이션 기반 신입사원 온보딩 웹앱**

---

## 2. Goal

### 문제 (Problem)
신입사원은 입사 초기에 회사 내 정보(인사제도, 시스템 권한, 업무 절차 등)를 파악하는 데 많은 시간과 에너지를 소모한다. 공채/수시 등 채용 형태에 따라 온보딩 경험의 질이 불균등하며, 체계적인 가이드 없이 개인이 정보를 직접 탐색해야 하는 경우가 많다. 또한 일반적인 온보딩 자료는 단방향 전달 방식이라 신입사원의 능동적 참여를 이끌어내기 어렵다.

### 해결책 (Solution)
Mega-Quest는 퀘스트 시스템과 가상 마스코트("트리니티 펫")를 활용한 모바일 웹앱으로, 신입사원이 온보딩 미션을 게임처럼 수행하며 자연스럽게 회사 정보를 습득하도록 유도한다. 채용 형태에 관계없이 동일한 표준화된 온보딩 경험을 제공하며, 포인트 보상과 펫 성장이라는 즉각적 피드백으로 참여 동기를 부여한다.

### 기대 효과 (Impact)
- 신입사원의 온보딩 정보 탐색 시간 단축
- 온보딩 미션 완료율 향상
- 신입사원 만족도 및 초기 몰입도 증대
- 채용 형태별 온보딩 품질 편차 해소

---

## 3. User Personas

### 페르소나 1: 신입사원 (Primary User)
- **이름:** 김신입 (26세)
- **상황:** 대기업 공채로 입사한 지 3일차. 사내 시스템이 생소하고 누구에게 무엇을 물어봐야 할지 막막함.
- **목표:** 빠르게 회사에 적응하고, 필요한 정보를 스스로 찾아 업무를 시작하고 싶음.
- **Pain Point:** 방대한 온보딩 자료, 선배 눈치 보기, 중복 질문에 대한 부담.
- **디바이스:** 주로 모바일(스마트폰)로 접속.

### 페르소나 2: HR 담당자 (Secondary / Demo Evaluator)
- **이름:** 박인사 (34세)
- **상황:** 해커톤 심사자로서 이 솔루션의 사업성과 UX를 평가.
- **목표:** 신입사원 온보딩 효율화 가능성과 확장 가능성 확인.
- **관심사:** 데모 완성도, 실제 도입 시 ROI, 유지보수 용이성.

---

## 4. High-Level User Journeys

### Journey 1: 온보딩 미션 탐색 및 수행
1. 신입사원이 모바일로 Mega-Quest에 접속.
2. 메인 대시보드에서 자신의 트리니티 펫(현재 1단계: 알)과 포인트 진행도 확인.
3. '퀘스트' 탭으로 이동하여 전체 미션 로드맵 확인.
4. '회의실 QR 체크인' 퀘스트를 선택.
5. QR 스캔 화면에서 '스캔하기' 버튼 클릭 (시뮬레이션).
6. '미션 완료' 팝업과 함께 5포인트 획득.
7. 메인 화면으로 돌아와 펫 성장 상태 확인.

### Journey 2: 퀘스트 로드맵 탐색
1. 신입사원이 퀘스트 탭에서 카테고리(HR-초보자 / 직군별-전직 / 일일·월간 / 입사 전 준비 / 첫 출근 / 필수 교육 / 조직 이해)를 선택.
2. 각 카테고리의 미션 목록과 보상 포인트, 완료/미완료 상태 아이콘 확인.
3. 잠긴(locked) 퀘스트와 수행 가능한(available) 퀘스트를 구분하여 다음 목표 설정.

### Journey 3: 펫 성장 확인
1. 퀘스트를 완료할 때마다 포인트 누적.
2. 포인트 임계치 도달 시 펫이 다음 단계로 진화(알 → 아기 → 메가 트리니티).
3. 진화 시 시각적 피드백(애니메이션 또는 상태 변화) 제공.

## 5. Business Requirements

### 기능 요구사항 (Functional Requirements)

**메인 대시보드 (Home)**
- 트리니티 펫을 포인트에 따라 3단계로 시각화 (1단계: 알, 2단계: 아기, 3단계: 메가 트리니티)
- 현재 포인트 및 다음 단계까지의 프로그레스 바 표시
- '오늘의 일일 퀘스트' 바로가기 버튼 제공

**퀘스트 로드맵 (Quest List)**
- 카테고리 탭: [HR-초보자], [직군별-전직], [일일/월간], [입사 전 준비], [첫 출근], [필수 교육], [조직 이해] — 총 7개 필터링
- 퀘스트 리스트 아이템: 제목, 보상 포인트, 상태 아이콘(완료/미완료/잠금)
- 퀘스트 선택 시 상세 화면 또는 미션 수행 화면으로 이동

**QR 미션 시뮬레이터**
- 카메라 뷰를 모사한 QR 스캔 UI
- '스캔하기' 버튼 클릭 시 `quest_id: 'meeting_room'` 상태를 `completed`로 변경
- 전역 포인트 +5 업데이트
- '미션 완료' 팝업(모달) 표시

**전역 상태 관리**
- 사용자 포인트 및 퀘스트 상태를 전역에서 관리 (Zustand 5 + persist middleware)
- 페이지 전환 시에도 상태 유지

**퀘스트 콘텐츠 (Mock Data)**
- 초보자 퀘스트: 인사제도 확인, 오피스 투어, 프로필 설정
- 전직(직군) 퀘스트: 슬랙/지라 권한 신청, MSP 기초 교육, CTU 소개
- 일일 퀘스트: 메일 확인, 전자결재 확인, 회의실 매너 준수
- 입사 전 준비: 입사 서류 제출, 보안 서약서 작성, 사내 계정 생성, 프로필 사진 등록, 자기소개서 작성
- 첫 출근: 사원증 수령, 장비 세팅, Wi-Fi/VPN 연결, 팀원 인사, 사무실 투어, 멘토/버디 확인
- 필수 교육: 정보보안, 성희롱 예방, 직장 내 괴롭힘 예방, 안전보건, 개인정보 보호, 사내 시스템 교육
- 조직 이해: 회사 비전/미션 퀴즈, 조직도 파악 (추가 예정)

### 비기능 요구사항 (Non-Functional Requirements)
- **Mobile-First:** 모든 UI는 모바일 뷰포트(375px 이상) 기준으로 최적화
- **반응형 디자인:** 태블릿 및 데스크톱에서도 정상 동작
- **성능:** 초기 로드 시간 3초 이내 (모바일 기준)
- **접근성:** 버튼 및 상호작용 요소에 적절한 레이블 제공
- **데이터 영속성:** MVP 범위 내에서 mock 데이터 사용; 실제 DB 연동 불필요
- **보안:** MVP이므로 인증/인가 시스템 불필요 (데모용 단일 사용자)

## 6. Success Metrics

| KPI | 목표 |
|---|---|
| 핵심 데모 시나리오 완주율 | 심사자 100%가 6단계 플로우 완료 가능 |
| QR 미션 완료 후 포인트 반영 정확도 | 오류 없이 +5P 반영 |
| 펫 진화 시각 피드백 | 포인트 임계치 도달 시 단계 전환 확인 가능 |
| 퀘스트 카테고리 필터 동작 | 7개 카테고리 탭 모두 정상 필터링 |
| 모바일 레이아웃 깨짐 없음 | iPhone 12 기준 레이아웃 정상 |

## 7. Out of Scope

- 실제 QR 코드 카메라 스캔 기능 (버튼 시뮬레이션으로 대체)
- 사용자 로그인/인증/권한 관리 시스템
- 실제 백엔드 서버 및 데이터베이스 연동
- 관리자(HR) 대시보드 또는 퀘스트 콘텐츠 CMS
- 푸시 알림 기능
- 다국어(i18n) 지원
- 포인트 초기화 자동화 및 스케줄링
- Gemini AI Q&A 연동 (선택 기능으로 별도 에픽 검토)
- 실제 슬랙/지라 시스템 연동

## 8. Business Value

**평가: High**

해커톤 맥락에서 이 에픽은 다음 이유로 높은 비즈니스 가치를 가진다:
- **차별화된 데모 임팩트:** 게이미피케이션 + 모바일 최적화 + 가상 마스코트 조합은 심사자에게 강한 인상을 남길 수 있음.
- **실제 문제 해결:** 신입사원 온보딩 비효율은 대부분의 기업이 공감하는 실존 문제로, 솔루션의 필요성이 분명함.
- **확장 가능성:** MVP를 기반으로 실제 HR 시스템 연동, AI Q&A, 관리자 기능 등으로 발전 가능.
- **빠른 구현 가능성:** Mock 데이터 + 프론트엔드 중심 구조로 해커톤 시간 내 완성 가능.
